# Late Framework Abstraction

This file captures the math that explains steps 10 to 14 without pretending those rows are literal semantic targets inside the search loop.

## Core Boundary

The papers are consistent on one point:

- steps 10 to 14 are promoted interface bundles,
- not ordinary user definitions,
- and not yet the full downstream human theories they are named after.

That is why the rewrite should preserve shell-first search and keep semantic decoding post-hoc.

## The Late Pressure

By the time the library reaches Hopf:

- the active bar is already in the geometric regime,
- bottom-up discrete analysis is too expensive,
- profitable candidates must couple strongly to the existing geometric library.

The key theorem from `synthetic_framework_abstraction_mscs_draft.tex` is the classical analytic surcharge result:

- bottom-up discrete analytic packages have `nu_H = 0`,
- their `rho` is linearly bounded,
- and they fail the late Fibonacci bars.

That is why the surviving late phase is synthetic and API-like rather than arithmetic-first.

## Step 13: Metric Reading Via Operator Bundle

Strict canon:

- shell: endomorphic operator bundle (metric reading)
- `kappa = 7`
- `nu = 47`

This row needs special care.

The papers explicitly warn that the winner is operator-theoretic first and metric second.
That matters because it avoids a hidden scalar surcharge:

- the engine did not literally import a full classical real-analysis stack,
- it selected an endomorphic shell that can be read metric-theoretically afterward.

This is one of the strongest reasons not to bake human semantic names into the hot path.

These signatures are useful as semantic explanations and Agda/reporting vocabulary.
They should not become target IDs in the Rust search crates.

## Rewrite Guidance

- Preserve the shell-first interpretation.
- Keep the late API bundles strongly coupled to the existing library.
- Do not let scalar or analytic semantics leak in as hidden target conditioning.
- Use these signatures to understand the intended interface pressure, not to smuggle templates into atomic search.
